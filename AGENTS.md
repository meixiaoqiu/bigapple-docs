# Big Apple Docs AI 工作规则

## 基本原则

- 默认使用中文说明修改内容和验证结果。
- 修改前先运行 `git status --short`，确认已有未提交改动。
- 优先维护目录清晰，不为单篇文档随意新增一级分区。
- 第一阶段只使用 Docusaurus 默认 classic 主题和 Markdown 文档。
- 不主动创建 MDX 页面、自定义 React 组件或额外 UI 框架。
- 不提交 `node_modules/`、`build/`、`.docusaurus/` 或其他生成缓存。

## 文档修改规则

- 普通说明文档可以按主题归入现有分区。
- 修改治理文书前，必须确认文本状态、版本、来源和治理流程记录。
- 不要把章程、决议、政策、规则或修订案当作普通说明文随意改写。
- 正式治理文书修改必须保留版本、来源、生效信息和修订历史。
- 未经过治理流程确认的文本，只能标记为草案、示例或说明，不得写成正式有效版本。

## 许可证与来源

- 非代码内容默认适用 `LICENSE` 中的 CC BY-SA 4.0。
- Docusaurus 配置、脚本、示例代码和构建相关代码默认适用 `LICENSE-CODE` 中的 AGPL-3.0-or-later。
- Fork、翻译、改编和镜像版本不得冒充“大苹果社区正式治理文书”“大苹果社区正式决议”或“大苹果社区正式规则”。

## 验证

- 修改文档站配置或导航后，至少运行 `npm run build`。
- 修改依赖后，运行 `npm install` 并检查生成的 `package-lock.json`。
- 命令失败时报告失败命令、关键输出和是否属于网络、权限或外部服务问题。
